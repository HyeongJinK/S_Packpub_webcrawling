Installing Marathon

Marathon is similar to traditional init systems. Here, for Mesos, it is a that helps in launching long applications.

Go to the Marathon tab. We can install it by running the following command:

sudo yum -y install marathon

And that's it, we are done with installing marathon. Perform this for the other server as well.

You can see the status as FAILURE, which is a good sign, remember? We will start the services later on:

 

Validating services

Let's now validate everything we have done up until now by testing if all the services are running.

To validate our masters, go to a master's tab and enter the following command:

sudo service mesos-master start

Then, to check the status, run the following command:

sudo service mesos-master status

You will now see what you have been waiting for:

Did you forget the ZooKeeper yet? Let's check that too. First, we start the service with the following command:

sudo service zookeeper start

Then, to check its status, run the following:

sudo service zookeeper status

You should see a similar active status to that shown in the preceding screenshot.

To validate slaves, run the following command on a slave's tab:

sudo service mesos-slave start

Then, to check the status, run the following command:

sudo service mesos-slave status

Marathon needs some more configuration first, so we will start its services later. We have installed Mesos for masters and slaves.
